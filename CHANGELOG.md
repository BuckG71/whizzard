## [Unreleased]

### Added
- **Stage 8 — Hermes adapter** (M1–M8 all shipped). Full end-to-end: adapter Protocol implementation, `active_capabilities()` surface (D-89, D-90), OneCLI-mediated credential injection with env-var fallback (D-134), `gateway.lock` preflight with stale-pid auto-recovery (D-87), `whiz hermes profile create` verb with `--clone-from` / `--no-clone` and `auth.json` exclusion (D-80, D-86), SIGTERM-based `wrap_up()` via `docker stop --time=<grace>`, HERMES_HOME bind mount with scoped UID parity (D-56, D-79), `docker/Dockerfile.hermes` derived image, manual smoke validation against Mac Studio Ollama via `host.docker.internal:11434`.
- **Stage 9 — in-cell MCP server (read-only subset).** `whiz_status`, `whiz_audit_self`, `whiz_emit_event`, `whiz_list_presets` (D-25, D-156). Launch-time snapshot + event-file write-back pattern; merged into host audit log at session_end.
- **Stage 10 — presets and CLI ergonomics.** Schema-versioned `presets.json` with omit-to-inherit override semantics (D-148 design pass). Brevity aliases (`whiz r`/`s`/`p`/`m`/`pr`) and smart defaults: bare `whiz` shows status, bare `whiz r` launches the most-recent preset. Bundled `hermes` and `shell` presets (D-101 personal-MVP defaults).
- **Stage 11 — harness integration recipes.** `docs/examples/<harness>/` shipped (D-161 — reframed away from the original Claude-Code-locked slash-commands bundle): `docs/examples/claude_code/` with skills shelling out to the OIQ CLI, `docs/examples/hermes/` with the Migrate recipe (`harnesses.json.example`, `config.yaml.snippet`, profile-cloning walkthrough). Root README "Using OIQ inside your agent harness" section.
- **Stage 12 — cross-adapter credential utility.** `whizzard/adapters/_credentials.py` with `fetch_secret(name)`: OneCLI first (D-134), host-env fallback. Credential source surfaced via `active_capabilities`.
- **Stage 13 — `oiq adjust` mid-session capability adjustment** (D-163). `whizzard/adjust.py` library + `whiz adjust <sid>` CLI: `--add-mount`, `--remove-mount`, `--extend`, `--allow-broad-mount`, `--yes`. TTY approval prompt shows a compact diff; narrowing-only adjusts skip the prompt. Stop+restart via the existing adapter `wrap_up()` path (D-27). Forward-compat hooks for Stage 14: pluggable `Approver`, library-shaped `adjust_session()`, `AGENT_DENIED_CHANGES` filter.
- **Stage 14 — agent-initiated capability requests** (D-165). In-cell MCP tools `whiz_request_mount` / `whiz_request_extend` / `whiz_check_request` drop request JSON into a per-session channel; the operator reviews/approves with `whiz requests {list,approve,deny}` which routes through `adjust_session(..., agent_initiated=True)` (the `AGENT_DENIED_CHANGES` filter blocks broad-mount / profile-change requests from the agent path). Pre-flight validation prevents an agent-requested mount that would need a broad-mount override from killing the session on a failed relaunch. `whiz status` shows a pending-request count.
- **Stage 15 — duration + idle-timeout enforcement** (D-166). `whizzard/enforcement.py` `monitor_and_enforce` — a single-threaded `Popen` poll loop in `run_shell` that gracefully stops the container on duration / idle hit. Hybrid idle detection: container resource activity (CPU, network, block I/O via `docker stats`) plus agent event-file / request-channel writes; either resets the idle clock. `oiq adjust --extend` wires through as a real `duration_override_seconds` = (cap − elapsed) + extension. `idle_timeout_seconds` added to the Profile schema. Pre-expiry warning fires once at a lead time before the cap as a `session_expiry_warning` audit-log event the agent reads via `whiz_audit_self`; `whiz status` shows live remaining time next to RUNNING. Session-end records `expiry_reason`.
- **Stage 15.5 — hot-restart of idle-ended sessions** (D-168, D-169). `whiz wake [<sid>]` verb: with a sid it wakes that specific session, bare it wakes the most-recent idle-ended-and-not-yet-woken session. Eligibility restricted to `expiry_reason == idle` so duration caps can't be circumvented. Permission set carries forward verbatim including any one-time `--allow-broad-mount` override (D-168 — the user-initiated-vs-not distinction, not host-CLI-vs-Discord, is what governs strip-vs-preserve). Missing mounts fail with a named-path error unless `--allow-missing-mounts` is passed to drop them. The `session_woken` audit-log event links old sid → new sid, and is the load-bearing signal that prevents repeated `whiz wake` from double-picking the same idle session. Verb name `wake` chosen over `resume`/`r` to avoid colliding with the Stage 10 preset-launch alias.
- **MVP+ Stages 19 + 20 (planned, OSS-launch prep).** Stage 19 — Packaging & Install (published Python distribution, execution-image distribution, clean-machine install verification, first-run config init). Stage 20 — Security Review & Hardening Audit (consolidated threat model, adversarial red-team test suite — one cluster per architecture.md invariant — fail-closed audit closing D-133, injection / command-construction audit, credential-handling audit, supply-chain scan in CI, independent reviewer pass).
- **`whizzard/` package restructured.** `cli.py` split into the `whizzard/cli/` package (per-subcommand modules: `_launch`, `_session`, `adjust`, `harnesses`, `hermes`, `image`, `mounts`, `preset`, `profiles`, `requests`, `sessions`). The CLI binary is `whizzard` / `whiz`.
- **In-cell MCP deployment** (D-167). `Dockerfile.hermes` `pip install`s `mcp` and `COPY`s `whizzard/mcp_server.py` in as a standalone script — the cell runs the MCP server without the full `whizzard` package present (which would expose the policy-layer implementation to a compromised agent — idle detection becomes trivially defeatable if `enforcement.py` is readable).
- **Integration smoke harness.** `tests/integration/` extended from 3 Stage-1 image probes to 22 tests covering: containment (Docker socket unreachable, capabilities dropped, network-off, read-only rootfs, non-root user, bind-mount isolation, hostile-symlink containment, read-only mount writes blocked), Stage 15 enforcement (duration + idle each stop real containers), the full `run_shell` launch path end-to-end, Stage 13 adjust primitives, in-cell MCP deployment (`mcp` SDK, `mcp_server.py` reachable, the D-167 invariant — `whizzard` package NOT in cell), MCP stdio protocol round-trip (which caught a real `tool_*` registration bug), Stage 8 Hermes binary + Ollama reachability. Shared `conftest` fixtures: `whizzard_base_image`, `whizzard_hermes_image`, `run_in_cell`, `launch_real_cell`, `ollama_reachable`.
- **Architectural support for non-interactive launch.** `build_run_argv` / `run_shell` gain `interactive: bool = True` (default unchanged); the smoke harness uses `False`. `monitor_and_enforce`'s `poll_interval` is now resolved at call time so the constant stays monkeypatchable.
- **D-162 — declarative `secrets:` field** in harness config for LLM-provider credentials; uses the same fetch_secret delivery as platform tokens. Plaintext credential values prohibited in harness configs.
- **Decisions log infrastructure.** Flat+tags schema (Type / Tags / Door Type / Decision / Rationale / Source / Status), curated canonical tag vocabulary, `scripts/validate_decisions.py` for schema + tag + reference integrity, `scripts/dx.py` lookup CLI. Length guidance tightened to keep entries succinct (most ≤100 words, substantive up to ~200) — the doc is for reviewers following the evolution, not a comprehensive log.
- **Dev tooling.** `pyproject.toml` configs for ruff (lint + import order + modernization) and mypy (phased adoption); `Makefile` shortcuts (`make check`, `make test`, `make integration`, `make coverage`); `.pre-commit-config.yaml` running validate-decisions + ruff + mypy; GitHub Actions CI workflow (`.github/workflows/ci.yml`) running lint + typecheck + test matrix (Python 3.11, 3.12) + decisions-validation on push and PR.
- **Documentation.** `docs/vision_and_strategy.md`, `docs/architecture.md`, `docs/mvp_build_plan.md` (Stages 1–18 + 15.5 + MVP+ 19–20), `docs/post_mvp_spec.md`, `docs/control_surface.md`, `docs/stage_validation.md`, `docs/decisions.md` (167 captured decisions), `docs/home_lab_deployment.md`, `docs/STAGE_8_BUILD_PLAN.md`, `docs/HANDOFF.md` (append-only log + mutable Current State header per D-150), `docs/similar_tools_research.html` (competitive landscape), `docs/reference/whizzard-architecture.html` (self-contained architecture overview), `docs/examples/` (per-harness integration recipes), `CLAUDE.md` (HTML-first output guidance for human-facing deliverables), `docs/internal/` (gitignored local-only docs area).

### Fixed
- **MCP tool registration names** (caught by the stdio smoke). `FastMCP` was registering each tool under its Python function name (`tool_whiz_status`, etc.) rather than the documented agent-facing name (`whiz_status`, etc.). Tools are now registered with an explicit `name=` matching architecture.md / decisions / the cooperation-layer surface. Unit tests missed this because they call the tool functions directly, bypassing FastMCP — exactly the class of wiring bug the integration smoke harness was built for.
