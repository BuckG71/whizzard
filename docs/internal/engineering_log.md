- **Stage 14 — agent-initiated capability requests** (D-165). In-cell MCP tools `whiz_request_mount` / `whiz_request_extend` / `whiz_check_request` drop request JSON into a per-session channel; the operator reviews/approves with `whiz requests {list,approve,deny}` which routes through `adjust_session(..., agent_initiated=True)` (the `AGENT_DENIED_CHANGES` filter blocks broad-mount / profile-change requests from the agent path). Pre-flight validation prevents an agent-requested mount that would need a broad-mount override from killing the session on a failed relaunch. `whiz status` shows a pending-request count.
- **Stage 15 — duration + idle-timeout enforcement** (D-166). `whizzard/enforcement.py` `monitor_and_enforce` — a single-threaded `Popen` poll loop in `run_shell` that gracefully stops the container on duration / idle hit. Hybrid idle detection: container resource activity (CPU, network, block I/O via `docker stats`) plus agent event-file / request-channel writes; either resets the idle clock. `oiq adjust --extend` wires through as a real `duration_override_seconds` = (cap − elapsed) + extension. `idle_timeout_seconds` added to the Profile schema. Pre-expiry warning fires once at a lead time before the cap as a `session_expiry_warning` audit-log event the agent reads via `whiz_audit_self`; `whiz status` shows live remaining time next to RUNNING. Session-end records `expiry_reason`.
- **Stage 15.5 — hot-restart of idle-ended sessions** (D-168, D-169). `whiz wake [<sid>]` verb: with a sid it wakes that specific session, bare it wakes the most-recent idle-ended-and-not-yet-woken session. Eligibility restricted to `expiry_reason == idle` so duration caps can't be circumvented. Permission set carries forward verbatim including any one-time `--allow-broad-mount` override (D-168 — the user-initiated-vs-not distinction, not host-CLI-vs-Discord, is what governs strip-vs-preserve). Missing mounts fail with a named-path error unless `--allow-missing-mounts` is passed to drop them. The `session_woken` audit-log event links old sid → new sid, and is the load-bearing signal that prevents repeated `whiz wake` from double-picking the same idle session. Verb name `wake` chosen over `resume`/`r` to avoid colliding with the Stage 10 preset-launch alias.

### Fixed
- **F-D-08 ripple bugs** (caught by the catch-up review). Microsecond-ISO timestamp format introduced by the Chunk D fix (`session_log._iso()` now writes `+00:00` offset) broke two `time.strptime` callers that expected the old `Z` suffix: `adjust.py:_session_elapsed_seconds` (made `oiq adjust --extend` silently reset the duration cap to "full original + extension" instead of "remaining + N", direct D-166 violation) and `cli/_session.py:_remaining_seconds` (made `whiz status` silently never render the time-remaining number for real sessions). Both now use `datetime.fromisoformat`, which handles both formats.
- **MCP tool registration names** (caught by the stdio smoke). `FastMCP` was registering each tool under its Python function name (`tool_whiz_status`, etc.) rather than the documented agent-facing name (`whiz_status`, etc.). Tools are now registered with an explicit `name=` matching architecture.md / decisions / the cooperation-layer surface. Unit tests missed this because they call the tool functions directly, bypassing FastMCP — exactly the class of wiring bug the integration smoke harness was built for.
