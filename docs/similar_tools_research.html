Whizzard landscape scan · 2026-05-22

The space is crowded but splits into four clusters. Only the highlighted row is a real competitor; the rest are adjacent.

ClusterExamplesHow it differs from Whizzard
Local harness-neutral sandbox wrappers scode, dagger/container-use, Anthropic sandbox-runtime Closest neighbors. They isolate but don't govern: no profiles, no human-readable mount list as the permission surface, no time-bounding, no behavioral layer. scode is the one to watch.
Harness-native sandboxing Claude Code sandboxed Bash / network Single-harness, and the layer Whizzard deliberately does not recreate. (A Claude Code sandbox bypass was disclosed May 2026 — good launch evidence.)
Cloud code-execution sandboxes E2B, Daytona, Modal, Northflank, Vercel Sandbox Cloud-first infra for AI products — not local governance for a developer's own agents. Possible future backend, not a competitor.
Enterprise governance / MCP gateways Permit.io, Kong, Cerbos, Permiso, MS Agent Governance Toolkit Uses Whizzard's vocabulary but targets org IAM / service-account fleets — explicitly not the v1 solo-dev audience.
Verdict: nothing occupies Whizzard's exact spot. The unoccupied ground is the combination: local-first + harness-neutral + mount-list-as-permission + named profiles + enforced time-bounding + the in-cell MCP cooperation layer + the planned Breaker. scode is the one tool close enough to encroach if it adds governance features.
