Architecture at a glance

Whizzard

Whizzard is a CLI tool that runs in your terminal. It builds a hardened Docker sandbox, with user-configured permissions and filesystem access scoping, in which the user can run third-party agent harnesses such as Hermes Agent (currently supported), NanoClaw (planned), OpenClaw (planned), etc.

The host is the control plane; the sandbox is the execution plane.

01Four lifecycle flows

Capabilities only ever change at well-defined moments. Each flow is host-driven — the agent can request, but never act on its own grant.
Launch whiz run · whiz r <preset>
1Resolve profile + mounts + adapter
2Run 3-tier safety validation
3Write session snapshot
4Launch sandbox → start enforcement monitor
Adjust / Wake mid-session change
1whiz adjust / wake → adjust_session
2Stop container
3Relaunch with new capabilities
4Harness state persists via bind mount
Agent request cooperation layer
1Agent calls whiz_request_mount / _extend
2Lands in /run/whiz/requests/
3Operator reviews whiz requests
4Approve → adjust_session
Enforce time-bounded sessions
1monitor_and_enforce polls the sandbox
2Duration cap or idle timeout hit
3Adapter wrap-up → SIGTERM
4Record expiry reason to audit log

02Harness adapters keep the core neutral

Whizzard decides what the sandbox can do; the adapter decides what runs inside it. Adapters are built from declarative harnesses.json entries.
Whizzard Coreharness-neutral — knows nothing about any specific harness
↔

↔

03Every mount target falls into one of three tiers

Every requested path is classified before launch. The tier — not the agent — decides whether a mount is allowed, gated, or refused.
⛔ Hard block — no override
/, $HOME, ~/.ssh, ~/Library, keychains, browser profiles, the Docker socket, the Whizzard config dir.
⚠ Override-required
Broad folders (~/Documents…), cloud-sync roots, parents of registered mounts. Needs --allow-broad-mount + a permitting profile — two gates. Every override is logged.
✓ Allowed
Registered mounts in mounts.json whose paths fall outside the above. The mount list is the permission model.
